В работе применено 3 патерна:
1. Структурный. Декоратор (Decorator) - Был выбран для создания динамического подключения дополнительных функциональностей. С его помощью реализовано создание требуемого количества участников.
2. Порождающий. Абстрактная фабрика (Abstract factory) - С помощью этого патерна реализована возможность создания нескольких видов соревнований. Благодаря патерну можно легко ввести дополнительный тип соревнований.
3. Поведенческий. Наблюдатель (Observer) - С помощью этого патерна реализована подписка наблюдателей на выбранное соревнование с их последующим уведомлением.
